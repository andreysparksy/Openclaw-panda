# Google Doc Format Reglament

## Purpose

This file defines how the final Google Doc should be structured and written.
The goal is not to dump raw observations, but to produce a readable working document with clear recommendations, examples, and ready-to-use rewritten variants.

## General formatting principles

The document must be:
- easy to scan
- split into clear blocks
- free from long unstructured walls of text
- written as a practical working document, not as an abstract report

The agent should separate:
- observation
- recommendation
- rationale
- example
- revised version

## Top-level document structure

Use this structure unless the owner explicitly asks for another format.

### 1. Заголовок документа

Format example:
- `Аналитика конкурентов / контента — <клиент> — <ниша>`

### 2. Вводный блок

Include:
- Клиент
- Ниша
- Площадка
- Дата
- Что анализировалось

### 3. Найденные кейсы / источники

List sources with short notes.

Format example:
1. Название — ссылка — короткая пометка
2. Название — ссылка — короткая пометка
3. ...

### 4. Что повторяется в кейсах

Highlight recurring patterns such as:
- офферы
- боли
- заходы
- механики
- CTA
- форматы посадки

### 5. Вот что должно быть по тексту в креативах

Use a numbered list:
1. ...
2. ...
3. ...

### 6. На какую посадочную страницу я рекомендовал бы вести

Use a numbered list and tie recommendations to the actual ad platform.
For VK Ads, keep recommendations realistic:
- лид-формы
- бот
- сообщение в группу / сообщество

### 7. На какую аудиторию запускать рекламу в первую очередь

Use a numbered list:
1. ...
2. ...
3. ...

### 8. Редакторские / маркетинговые рекомендации

For each issue, structure the recommendation in blocks.

## Mandatory recommendation format

When analyzing text, content, creatives, or channel materials, use this format:

### Проблема
Short explanation of what is weak or underperforming.

### Что изменить
Short operational recommendation.

### Исходный вариант
Original text or structure.

### Вариант с правками
Rewritten version.

This is the preferred format when the task involves content improvement, message improvement, post rewrite, or conversion-focused edits.

## Channel-specific formatting

If the analysis covers multiple channels, split recommendations by channel instead of mixing them.

### Telegram
Focus on:
- headline strength
- first 2–3 lines
- structure readability
- CTA and transition into lead capture

### Instagram
Focus on:
- visual-first structure
- carousel logic
- first 2 seconds of attention
- caption clarity
- competition with visual content

### VK / VK Ads
Focus on:
- ad mechanics
- practical landing format
- message angle
- first audience segments

## Writing style requirements

Recommendations must be operational, not vague.

Avoid phrases like:
- `можно улучшить текст`
- `стоит усилить оффер`

Prefer phrases like:
- `усилить заголовок, вынеся ключевую выгоду в первые 2 секунды внимания`
- `переработать кейс из отчётного формата в сюжет с героем`
- `встроить незавершённую пользу в основной контент для нативного перехода в лид`

## Document layout rules

- Start each major section with a visible heading.
- Use numbered or bulleted lists where appropriate.
- Separate long examples with blank lines.
- Always label `Исходный вариант` and `Вариант с правками` explicitly.
- Do not mix recommendations for different channels in one paragraph.
- Prefer conclusion first, example second.

## Quality bar

A strong final document should include:
- concrete recommendation
- rationale for why it matters
- original version
- revised version
- separation by channel when relevant
- practical lead-generation focus
- easy-to-implement next steps
