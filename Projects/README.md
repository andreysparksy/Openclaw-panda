# Projects

Здесь хранится структурированная информация по клиентам.

## Базовое правило
- 1 клиент = 1 отдельная папка внутри `Projects/`
- в папке клиента лежат бриф, ссылки, операционные заметки и другие материалы
- новые данные по клиенту сначала раскладываются по смыслу, затем сохраняются сюда

## Правило для новых клиентов
Когда приходит новый клиент:
1. создаём для него новую папку внутри `Projects/`
2. оформляем информацию по регламенту
3. за основу берём структуру предыдущего клиента как шаблон
4. не смешиваем брифовую информацию, ссылки, операционные этапы и сырые вводные в одну кучу

## Регламент оформления
Минимум, который должен быть в папке клиента:
- кто клиент
- компания / бренд
- ниша
- география
- услуги
- контакты
- ссылки на брифы / таблицы / соцсети
- позиционирование
- что уже собрано
- что требует уточнения

## Если информации не хватает
Если по клиенту не хватает данных, агент не додумывает их сам, а прямо пишет, что ещё нужно получить.
Пример формулировки:
- не хватает контактов
- не хватает ссылки на группу
- не хватает позиционирования
- не хватает информации по оплате
- не хватает оффера / преимуществ / кейсов
